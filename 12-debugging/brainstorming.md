# Good habits for avoiding bugs
- watch your "end"s (close things as soon as you open them)
- watch your brackets
- use meaningful and consistent names (for methods, variables, classes)
- good indentation
- watch your spelling
- run your code frequently (make small changes)
- be mindful of data types (esp arguments, return values)
- use the right data type for what you're trying to do


# Techniques for dealing with bugs
- read the error message
- google the error message
- ask for help
- pair programming
- que
- use a debugger
- double check your spelling (look for typos)
- print out the values of variable
- print out a general message (sanity check)
- turn it off and on again
